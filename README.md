# 🚀 OnliSharpTeam IDE — AI Collaborative IDE

**Платформа реального времени для совместного программирования с AI-ревьюером.** Проект разработан в рамках хакатона «Разум 2.0». 

Цель проекта — создание полноценной веб-IDE для одновременного написания кода с умным AI-ассистентом, способным на лету разрешать синтаксические конфликты и проводить ревью кода.

---

## 🌟 Ключевые особенности и архитектурные решения

* **Совместное редактирование (CRDT) и WebSockets:** Полная синхронизация состояния редактора, глобальная история изменений (Undo/Redo) и отрисовка чужих курсоров в реальном времени на базе **ASP.NET Core SignalR**.
* **In-Memory Компиляция (Песочница):** Вместо классического и медленного подхода с поднятием Docker-контейнеров, мы реализовали динамическую компиляцию C# кода прямо в памяти сервера с использованием **Roslyn (CSharpCompilation)**. Это обеспечивает изоляцию, безопасность и скорость выполнения скриптов в миллисекундах.
* **AI-Ассистент (GigaChat API):**
  * **Code Review:** Интеллектуальный анализ кода на наличие антипаттернов и ошибок по запросу.
  * **Разрешение конфликтов:** Нейросеть автоматически анализирует две конфликтующие версии кода (когда два студента правят один участок) и предлагает оптимальный `merge`-вариант.
* **Бессерверная авторизация (Stateless):** Защита комнат и пользователей реализована через JWT-токены.

---

## 🛠 Технологический стек
* **Бэкенд:** C#, ASP.NET Core 9, Entity Framework Core (SQL Server), SignalR.
* **Фронтенд:** Vanilla JS, HTML, CSS, Monaco Editor (ядро VS Code).
* **AI Интеграция:** GigaChat API.
* **Инфраструктура:** ngrok (для туннелирования и тестирования WebSockets).

## ⚙️ Инструкция по локальному запуску

Для корректной работы совместного редактирования по сети требуется запуск локального сервера и проброс туннеля через `ngrok`.

### 1. Требования к окружению (Prerequisites)
* Установленный **.NET 9 SDK**.
* Установленный **SQL Server**.
* Установленная утилита **ngrok** (необходима для работы WebSockets (SignalR) через защищенный HTTPS-туннель).

### 2. Настройка Базы Данных и Конфигов
1. Склонируйте репозиторий и откройте терминал в корневой папке бэкенда (где находится файл `HackathonIde.csproj`).
2. Убедитесь, что в файле `appsettings.json` указаны актуальные ключи для `GigaChat`.
3. Примените миграции Entity Framework для создания таблиц в БД:
   ```Update-Database```
### 3. Запуск Бэкенда и Туннеля
Запустите бэкенд-сервер:

По умолчанию сервер запустится на http://localhost:5110.

Откройте новый терминал (не закрывая бэкенд) и запустите ngrok для проброса порта 5110:
ngrok http 5110
В консоли ngrok появится HTTPS-ссылка (вида https://<ваш_уникальный_ид>.ngrok-free.dev). Скопируйте её.

### 4. Связывание Фронтенда с Бэкендом (ВАЖНЫЙ ШАГ)
Поскольку ngrok при каждом запуске генерирует новую ссылку, фронтенду нужно указать актуальный адрес сервера:

Откройте файл фронтенда: wwwroot/app.js.

На самой первой строке найдите константу NGROK_URL.

Вставьте туда скопированную ссылку из ngrok:

const NGROK_URL = "https://<ваш_уникальный_ид>.ngrok-free.dev";
Сохраните файл app.js.

### 5. Использование приложения
Обход экрана безопасности ngrok: Откройте вашу ссылку ngrok (https://<ваш_уникальный_ид>.ngrok-free.dev) в браузере. Вы увидите страницу с предупреждением от сервиса ngrok. Обязательно нажмите синюю кнопку "Visit Site". Это необходимо, чтобы API-запросы и WebSockets не блокировались заглушкой.

Пройдите регистрацию / авторизацию.

Создайте новую сессию (комнату) и установите для неё пароль.

Для проверки совместной работы: передайте ссылку на index.html (если хостите фронт) или попросите напарника открыть проект, передайте ему пароль от комнаты, и вы увидите курсоры друг друга в реальном времени!
